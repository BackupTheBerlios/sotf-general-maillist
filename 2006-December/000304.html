I think it is a great thing that SOTF has a CSS based layout. However, I 
admit, that I have some comments on it:
- I will not comment on the colours as those may be changed easily now, I 
hope :-) There are some nice parts of the design, the content links for 
example, and some lists with the action-icons.
- The original search interface had more opportunities than this, for 
example one could search for an interval of broadcast date (from-until)
- The station list could have an expanded view where all icons are shown
- The layout of the topics list is not fortunate, as Laszlo wrote. Too much 
sscrolling will be the result.
- The station view is compact, gives a good overview for me, again at the 
price of hiding all icons.
- The main problem of the program view is that too much scrolling is needed 
to get to the content. In the design made by our Italian guest student, she 
put the metadata and content links side-by-side, so the content could be 
reached without scrolling. In case of video, now there is the preview part, 
but the preview part could contain the link to start watching the content. 
It would be nice to have a preview part for audio as well :-) Some audio 
curves maybe? :-)
- There are problems with the layout as well. With large fonts it very 
easily breaks apart.

I know that making pure CSS design is very tricky, but if this goes against 
usability, then the principles of pure CSS could be alleviated...
