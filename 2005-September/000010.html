problem is solved. debian offers a cupple of tools for subversion:

debian sarge or sid:
subversion - advanced version control system (aka. svn)
subversion-tools - assorted tools related to Subversion (aka. svn)
cvs2svn - Convert a cvs repository to a subversion repository
esvn - frontend for the Subversion revision system written in Qt
esvn-doc - documentation for esvn
python-svn - A(nother) Python interface to Subversion
rapidsvn - A GUI client for subversion

>> it seems rather simple to convert the repository to SVN and we would 
>> really wanna make the switch because revisioning is simpler in SVN 
>> (each commit increases revision on the whole tree, making it easier to 
>> go a step back or to keep up with changes), branching is simpler (uses 
>> directories where you can simply copy and merge between branches and 
>> it's easier to read because no meta dimension for branches is used, 
>> just plain folders)
>>
>> For the node admins this means:
>> - you have to backup your node installation
>> - delete code and www directory in your node
>> - checkout code and www directory from SVN
>> - copy config.inc.php from backup
>> - reapply modifications you did to either the files in code or www 
>> directory
