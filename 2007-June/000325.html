hi all,

after a long long time, i'm happy to tell you that i restored the  
dornbirn node and i lost NO data.
now both my nodes are running again, while i still have problems to  
sync to the network.

some background:
i had to kill my machine after being hacked 5(!) times in a half  
year. some very nasty things happend, allowed by PHP :-(
mass mails were sent, a rootkit was installed and so on. I tracked  
the problem down to PHP, but i couldn't figure out if the problem was  
caused by SOTF (xmlrpc flaws maybe or other security hole?) or  
PHPmyAdmin.

Either way i was force to rethink my whole concept of hosting PHP and  
SOTF applications and came up with this setup:

- standard xserver with OS X server running only out-of-the-box  
apache 1.3, NO PHP installed. this server acts as a proxy to several  
other servers that are located in a private network, not reachable  
from the outside. the virtual hosts run on the apache on the xserver  
accessing the other servers via mod_rewrite rules proxied through  
apache.

- got wordpress, sotf and other working this way

- provides some better level of security as exploits can still get  
through the proxy to the php webapps, but if something gets nasty  
again, i can turn off this server without loosing my complete  
functionality not related to PHP (zope, streamingserver, ...)

ok, this works for the website, but now i have a problem with the  
syncing:
"No access: this IP is not from neighbour 667!"

anyone has an idea how i can work around this? Maybe we can tweak the  
verification process for xmlrpc to allow my setup? andras, any ideas?

greetz from vorarlberg,
