> there is a problem with clients using itunes as player. if they  
> want to
> listen (click on speaker icon) to a broadcast with prepended jingle
> (e.g. of the station) itunes plays only the jingle, but not the
> following audiofile containig the boradcast.
>
> i suppose this happens because of missing EXTM3U tags ('#EXTM3U' and
> '#EXTINF:'), maybe also the carriage return acts a part (if you  
> open the
> m3u under windows you see all urls of audiofiles in a single row).

the problem is itunes itself and it's a problem by design :-(
itunes can read m3u files, which is perfectly fine, but it imports  
all the files (or urls) contained in it right into its library -  
which is bad. as a sidenote: both files are imported into the  
library, not only the jingle, but as it breaks the m3u playlist by  
importing all into the library the order of the playlist breaks and  
after the jingle some other songs from your library will be played,  
the other file from the m3u file will dissappear somewhere in the big  
library (especially if you have 4000 songs in it ;-) )

for mac there was an attempt 1 or 2 years ago with a little freeware  
program which hooked into the m3u opening process and created  
automatically a playlist inside itunes with all the files from the  
m3u file. unfortunately this never worked good enough for me and its  
a mac only solution :-(

> seems that itunes is the only player affected with this troubles.
> unfortunately it's a rather rather widely spread player.

see my comment at the beginning :-D
time changes some things i guess ;-)
