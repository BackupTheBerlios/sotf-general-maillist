Great! We have a by now quite complex database structure and we lack  
proper treatment of audio metadata which can occur as timed elements  
of tracks within video clips or playlists. In terms of the two  
projects collaborating / integrating on a technical level I can think  
of two basic directions:

     1) We link the separate video and audio databases using postGres  
features, but keeping the databases separate.

     2) We integrate the two schemas learning from one and using it  
to extend the second.

There is an additional requirement from out point of view which is  
looking at the future technical platform to build web applications  
around the database structure. At the moment we have a variety of web  
services written in a combination of perl and php with interact with  
the data model, and rich interfaces written in Revolution which  
operate AJAX style. The aim is to extend this through user testing  
within a flexible design and requirements led process until these  
requirements stabalise and then port the interfaces to full AJAX  
based deployment where appropriate.

This means looking at modern MVC style web application frameworks of  
which there are a number (Ruby on Rails getting the most press at the  
moment), and looking to use the built in object relational mapping  
tools present in such frameworks to create the web application  
framework. This MVC framework can then then be directed to call the  
existing web services and views created in the first stage  
(regardless of which language the web services are created in).
